## Overview

Revision methods allow you to modify, update, and manage the lifecycle of memory records. These methods support versioning, forking, merging, and salience adjustment.

All revision operations require an `actor` (who is making the change) and a `rationale` (why the change is being made) for audit and provenance tracking.

## Supersede

Replace an old memory record with a new version, maintaining lineage.

### Request Parameters

<ParamField path="old_id" type="string" required>
  ID of the record being superseded
</ParamField>

<ParamField path="new_record" type="bytes" required>
  JSON-encoded MemoryRecord that replaces the old one
</ParamField>

<ParamField path="actor" type="string" required>
  Identifier of the actor performing the supersede operation
</ParamField>

<ParamField path="rationale" type="string" required>
  Explanation for why the record is being superseded
</ParamField>

### Response

<ResponseField name="record" type="bytes" required>
  JSON-encoded MemoryRecord representing the newly created version
</ResponseField>

### Example

<Tabs>
  <Tab title="Go">
    ```go
    newRecord := MemoryRecord{
        Type:    "observation",
        Content: map[string]interface{}{
            "subject":   "user:jane",
            "predicate": "role",
            "object":    "senior_engineer",
        },
    }
    newRecordBytes, _ := json.Marshal(newRecord)
    
    req := &membranev1.SupersedeRequest{
        OldId:      "rec-old-123",
        NewRecord:  newRecordBytes,
        Actor:      "agent-1",
        Rationale:  "User was promoted from engineer to senior engineer",
    }
    
    resp, err := client.Supersede(ctx, req)
    if err != nil {
        log.Fatal(err)
    }
    
    var updatedRecord MemoryRecord
    json.Unmarshal(resp.Record, &updatedRecord)
    ```
  </Tab>
  
  <Tab title="Python">
    ```python
    import json
    
    new_record = {
        "type": "observation",
        "content": {
            "subject": "user:jane",
            "predicate": "role",
            "object": "senior_engineer"
        }
    }
    
    req = membrane_pb2.SupersedeRequest(
        old_id="rec-old-123",
        new_record=json.dumps(new_record).encode(),
        actor="agent-1",
        rationale="User was promoted from engineer to senior engineer"
    )
    
    resp = stub.Supersede(req)
    updated_record = json.loads(resp.record)
    ```
  </Tab>
</Tabs>

### Use Cases

- Updating user profile information
- Correcting erroneous data
- Reflecting state changes over time

</ParamField>

</ResponseField>

## Provenance and Audit Trail

All revision operations maintain a complete audit trail:

- **Actor**: Who made the change
- **Rationale**: Why the change was made
- **Timestamp**: When the change occurred
- **Operation**: What type of revision was performed

This enables:
- Debugging memory behavior
- Understanding decision lineage
- Compliance and accountability
- Multi-agent coordination

## Best Practices

1. **Always provide meaningful rationales**: This helps with debugging and understanding system behavior
2. **Use supersede for corrections**: Maintains lineage while updating information
3. **Use fork for experiments**: Allows trying alternatives without losing original
4. **Reinforce successful patterns**: Help the system learn what information is valuable
5. **Contest rather than retract when uncertain**: Allows for multiple perspectives
6. **Track actor IDs consistently**: Enables per-agent or per-user analysis
